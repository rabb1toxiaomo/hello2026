# 贡献指南

感谢你对 **hello2026** 项目的关注！我们欢迎所有形式的贡献。

## 🤝 如何贡献

### 📝 报告问题

如果你发现了 bug 或有功能建议，请：

1. 检查 [Issues](https://github.com/rabb1toxiaomo/hello2026/issues) 中是否已有相关问题
2. 如果没有，请创建新的 Issue，详细描述问题或建议
3. 使用 Issue 模板，提供尽可能多的信息

### 💻 提交代码

1. **Fork 本项目**
   ```bash
   # 在 GitHub 上 Fork 项目
   ```

2. **克隆你的 Fork**
   ```bash
   git clone https://github.com/your-username/hello2026.git
   cd hello2026
   ```

3. **创建特性分支**
   ```bash
   git checkout -b feature/AmazingFeature
   ```

4. **进行更改并提交**
   ```bash
   git add .
   git commit -m 'Add some AmazingFeature'
   ```

5. **推送到你的 Fork**
   ```bash
   git push origin feature/AmazingFeature
   ```

6. **开启 Pull Request**
   - 在 GitHub 上创建 Pull Request
   - 使用 PR 模板，详细描述你的更改
   - 等待代码审查

## 📋 代码规范

### 代码风格
- 使用有意义的变量和函数名
- 添加必要的注释，特别是复杂逻辑
- 保持代码简洁，避免过度复杂
- 遵循现有的代码风格和格式

### JavaScript 规范
- 使用 `const` 和 `let`，避免 `var`
- 使用箭头函数（如适用）
- 使用模板字符串
- 保持函数简洁，单一职责

### 提交信息规范
- 使用清晰的提交信息
- 格式：`类型: 简短描述`
- 类型：`feat`（新功能）、`fix`（修复）、`docs`（文档）、`style`（格式）、`refactor`（重构）、`test`（测试）、`chore`（构建）

示例：
```
feat: 添加明信片生成功能
fix: 修复祝福语轮播不切换的问题
docs: 更新 README 中的部署说明
```

## 🛠️ 开发环境设置

1. **克隆仓库**
   ```bash
   git clone https://github.com/rabb1toxiaomo/hello2026.git
   cd hello2026
   ```

2. **安装依赖**
   ```bash
   npm install
   ```

3. **启动开发服务器**
   ```bash
   npm run dev
   ```

4. **访问应用**
   打开浏览器访问 `http://localhost:2026`

## 🧪 测试
